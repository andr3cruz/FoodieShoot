Behind the scenes


TensorFlow 2.0


We use TensorFlow 2.0 to allow us to use the TF lite API to convert our model to a compact and fast android compatible servable model

The data


To build the model we collected food data (images) and assembled a dataset, you can get it Here

Building the model for transfer learning


We use TensorFlow 2.0's high-level Keras API to quickly build our image classification model. For transfer learning we started using InceptionV3 model however, since our goal is to build a portable android model, we moved to a MobileNet model, this kind of models are designed to be resource friendly and quick.
Transfer learning speeds up training by enabling us to reuse existing pre-trained image classification models, only retraining the top layer of the network that determines the classes and image can belong to.
